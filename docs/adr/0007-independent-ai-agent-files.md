# ADR 0007 — Independent AI Agent Files (No Symlinks)

## Status
Accepted

## Context
holistic-ai originally generated AGENTS.md and symlinked CLAUDE.md and GEMINI.md to it. All three AI agents (Claude, Gemini, Codex) read identical content. This gave no agent a distinct epistemic surface and made it impossible for files to diverge — Codex couldn't accumulate implementation notes without overwriting what Claude and Gemini read, and Gemini couldn't write documentation-grade prose without replacing terse agent instructions.

A debate architecture requires each agent to argue from partial, distinct context. Shared content collapses the debate into a single perspective.

## Decision
CLAUDE.md, GEMINI.md, and QWEN.md are independent generated files — never symlinks.

- **AGENTS.md**: generated by holistic-ai on first run; Codex-owned thereafter (reads + writes implementation notes; holistic-ai reads but does not overwrite on re-run)
- **CLAUDE.md**: holistic-ai generates as a copy of AGENTS.md; refreshed on every holistic-ai re-run (not by Codex)
- **GEMINI.md**: Gemini generates from AGENTS.md + all domain docs; documentation-grade prose; holistic-ai does not touch it after initial generation
- **QWEN.md**: synthesized from GEMINI.md + domain docs + AGENTS.md in Traditional Chinese; read by Qwen (observer role)

## Considered Options

1. **Keep symlinks** — simple, single source of truth; rejected because it prevents divergence and collapses the debate
2. **Symlink CLAUDE.md only, generate GEMINI.md** — partial solution; rejected because CLAUDE.md still can't diverge from AGENTS.md as Codex writes to it
3. **All independent** (chosen) — maximum epistemic separation; each agent owns its file; debate architecture viable

## Consequences

- holistic-ai Step 8 changes from "create symlinks" to "copy AGENTS.md to CLAUDE.md + invoke Gemini to generate GEMINI.md"
- link-ai-agents skill no longer applies to CLAUDE.md or GEMINI.md — those are generated, not linked
- Re-running holistic-ai is safe: only CLAUDE.md is overwritten; AGENTS.md, GEMINI.md, QWEN.md are not
- Migration required for existing repos: remove existing CLAUDE.md/GEMINI.md symlinks, generate fresh independent files
